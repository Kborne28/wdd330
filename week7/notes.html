Notes from JavaScript: Novice to Ninja: Chapter 13: Ajax

Ajax is a technique that allows web pages to communicate asynchronously with a server, and it dynamically
updates webpages without having to reload. This enables data to be sent and received in the background, as well as
portions of a page to be updated in response to user events, while the rest of the program continues to run.



Clients and Servers

The internet is seperated into two parts: Clients and Servers.
Clients requests a web page from a server, and the server processes and sends that request.
JavaScript was designed to be a client-side scripting language, meaning that it rans locally
in the browser (the client side/part), adding dynamic features to the web page that was returned from the server.
Ajax allows JavaScript to request resources from a server on behalf of the client.
It is usually requested using JSON data or html.
In order to use Ajax, a server-side language is required to do so. Languages such as:
PHP
Ruby
Node.js
.NET

What is Cross-Origin Resource Sharing?

Cross-Origin Resource Sharing (CORS) is an HTTP-header based mechanism that allows a server
to locate any other origins (domain, scheme, or port) than its own from which a browser should permit
loading of resources. CORS also relies on "preflight" requests used by browsers to see if the server
will host the cross-origin request. In that request, the browser sends headers that indicate
the HTTP method and headers that will be used in the actual request.

What requests use CORS?
Invocations of the XMLHttpRequest or API's
Web Fonts(for cross-domain font usage in @font-face within CSS)so that servers can deploy
TrueType fonts that can only be cross-site loaded and used by web sites that are permitted to do so.
WebGL Textures
CSS Shapes from images

The Fetch API

Fetch API is the current living standard for requesting and sending data asynchronously across a network.
It streamlines a number of concepts that had become cumbersome when using the XMLHttpRequest object.
How does it work?

The Fetch API proveides a global fetch() method that only has one mandartory argument, which is to
fetch the resource that was requested. Here's an example:

fetch('https://example.com/data')
.then(//code that handles the response) .catch(//code that runs if the server returns an error)

The fetch() returns a promise that resolves to the response from the URL that was provided as an argument. The .then handles
handles that response, and the .catch statement helps deal with any errors that may occur.

The Reponse

The Reponse interface deals with the object that's returned when the promise is fulfilled.
Example:

const url = 'https:example.com/data'; fetch(url) .then((response) => { if(response.ok) { return response; } throw Error(reponse.statusText); }) .then(response => // do something with response) .catch(error => console.log('There was an error!'))

Headers Interface

HTTP Headers are used to pass on any additional info about a request or response.
The Fetch API introduced a Headers interface, which can be used to create a headers object.
A Headers object has the following properties:
has(): Can be used to check if the headers object contains the header provided as an argument.

headers.has('Content-Type');
true;
get(): Returns the value of the header provided as an argument
headers.get('Content-Type');
'text/plain';
set(): Can be used to set a value of an already existing header, or create a new header
with the value provided as an argument if it does not already exist.

headers.set('Content-Type', 'application/json');
delete(): Removes the header provided as an argument.
headers.delete('Accept-Encoding');
